01 / CONTEXT & PROBLEM

Analytics scripts rot. Engineered pipelines don't.

Most personal analytics projects are a notebook that worked once. This project's real objective was engineering discipline on a small, honest dataset: strict typing, validated inputs, incremental state, tests that gate every change, and automation that keeps the output current without a human in the loop.

03 / KEY DECISIONS & RESULTS

Discipline over dataset size

Strict mypy + pydantic everywhere

Every record and config passes typed validation; an upstream format change fails loudly at the boundary instead of corrupting history silently.

Modular stages behind one Typer CLI

Collect, validate, analyze and publish are separate testable modules; the CLI composes them so CI and a human run exactly the same path.

Statistics, honestly framed

Frequency and recency views describe historical draws — the dashboard never claims predictive power over a random process. Framing outputs truthfully is part of the job.

04 / WHAT I'D DO DIFFERENTLY

Graduate the storage layer.

Incremental CSV is transparent and diff-able, but as history grows I'd move to DuckDB or SQLite for typed queries and faster aggregates — keeping the CSV export as the human-readable artifact rather than the system of record.
